

PS7 — Study Chain Code in Hyperledger Fabric
Aim: Write and deploy a simple chaincode (smart contract) on a local Hyperledger Fabric test network,
then invoke and query it.
##  Step 1: Install Prerequisites
Open terminal and install all required tools:
Install Node.js v18 (LTS):
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v
npm -v
Install Docker and Docker Compose:
sudo apt-get update
sudo apt-get install -y docker.io docker-compose
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker $USER
⚠ After adding yourself to the docker group, log out and log back in for changes to take effect.
Install Git and curl: sudo apt-get install -y git curl
Install Go (required by Fabric):
wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.0.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
go version
✔ Should show go version go1.21.0
##  Step 2: Download Hyperledger Fabric Samples
cd ~
curl -sSL https://bit.ly/2ysbOFE | bash -s -- 2.5.0 1.5.7
This downloads fabric-samples folder with test-network and all binaries. It takes a few minutes.

ls fabric-samples
✔ Should show folders including test-network, bin, chaincode etc.
 Step 3: Add Fabric Binaries to PATH
echo 'export PATH=$PATH:~/fabric-samples/bin' >> ~/.bashrc
echo 'export FABRIC_CFG_PATH=~/fabric-samples/config' >> ~/.bashrc
source ~/.bashrc
Verify: peer version
✔ Should show Fabric version info
 Step 4: Start the Test Network
cd ~/fabric-samples/test-network
./network.sh down
./network.sh up createChannel -c mychannel -ca
This starts 2 peer nodes, 1 orderer, creates a channel called mychannel. Takes 1–2 minutes.
✔ Should show Channel 'mychannel' joined
##  Step 5: Create Chaincode Folder
mkdir -p ~/fabric-samples/chaincode/studychain/javascript
cd ~/fabric-samples/chaincode/studychain/javascript
##  Step 6: Initialize Node Project
npm init -y
npm install fabric-contract-api fabric-shim
✔ Should show node_modules created
##  Step 7: Create Chaincode File
nano studychain.js
##  Step 8: Paste This Chaincode
'use strict';

const { Contract } = require('fabric-contract-api');

class StudyChain extends Contract {

async initLedger(ctx) {

console.info('Ledger initialized');
## }

async addStudent(ctx, studentId, name, course) {
const student = {
studentId,
name,
course,
docType: 'student'
## };
await ctx.stub.putState(studentId, Buffer.from(JSON.stringify(student)));
return JSON.stringify(student);
## }

async getStudent(ctx, studentId) {
const data = await ctx.stub.getState(studentId);
if (!data || data.length === 0) {
throw new Error(`Student ${studentId} does not exist`);
## }
return data.toString();
## }

async updateCourse(ctx, studentId, newCourse) {
const data = await ctx.stub.getState(studentId);
if (!data || data.length === 0) {
throw new Error(`Student ${studentId} does not exist`);
## }
const student = JSON.parse(data.toString());
student.course = newCourse;
await ctx.stub.putState(studentId, Buffer.from(JSON.stringify(student)));
return JSON.stringify(student);
## }

async deleteStudent(ctx, studentId) {
const data = await ctx.stub.getState(studentId);
if (!data || data.length === 0) {
throw new Error(`Student ${studentId} does not exist`);
## }
await ctx.stub.deleteState(studentId);
return `Student ${studentId} deleted`;
## }
## }

module.exports = StudyChain;

Save and exit: Ctrl+O, Enter, Ctrl+X

 Step 10: Deploy Chaincode to Network
Go back to test-network:
cd ~/fabric-samples/test-network
Deploy the chaincode:
./network.sh deployCC \
-ccn studychain \
## -ccp ../chaincode/studychain/javascript \
-ccl javascript \
-c mychannel
This compiles, packages, installs, and commits the chaincode. Takes 1–2 minutes.
✔ Should show Chaincode definition committed on channel 'mychannel'

## Simple Flow
## 1. Install Node.js, Docker, Go
- Download Fabric samples
- Start test-network
- Write chaincode (studychain.js)
- Deploy chaincode to channel
- Set peer environment variables
- Invoke functions (add, update, delete)
- Query ledger
- Shut down network
## Command Reference Table

Command Why it is used
./network.sh up createChannel Starts Fabric network and creates a channel
./network.sh deployCC Packages and deploys chaincode to the channel
export CORE_PEER_* Sets peer connection and identity variables
peer chaincode invoke Writes data to the ledger (add/update/delete)
peer chaincode query Reads data from the ledger without a transaction
npm install fabric-contract-api Installs Fabric SDK to write chaincode in JS
./network.sh down Stops all Docker containers and cleans up
initLedger Initializes the chaincode state on the ledger
addStudent Stores a student record on the blockchain
getStudent Retrieves a student record by ID
updateCourse Updates the course field of an existing student
deleteStudent Removes a student record from the ledger
